Frequently Asked Questions

Why there is no *BSD support?

HyperCPU ISA is known to fail building from sources or throw exceptions in unexpected moments under *BSD systems (i. e. FreeBSD — building is successful, but hcemul throws std::exception when working with console I/O; OpenBSD — EPIC FAIL). Furthermore, nobody asked us for it.

Porting other HyperCPU Project utilities to *BSD systems makes no sense if HyperCPU ISA is not available on them.

Why there is no Windows support?

To make a long story short: HyperCPU utilities use GNU/Linux system API functional which may be hard or even impossible to (re-)implement using Windows API.

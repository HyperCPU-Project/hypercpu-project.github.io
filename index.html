Frequently Asked Questions


Why there is no *BSD support?

HyperCPU ISA is known to fail building from sources or throw exceptions in unexpected moments under *BSD systems. Porting other HyperCPU Project utilities to *BSD systems makes no sense if HyperCPU ISA is not available on them. It is also pointless because nobody currently asked us for it.


Why there is no Windows support?

To make a long story short: HyperCPU Project utilities use GNU/Linux system API functional which may be hard or even impossible to (re-)implement using Windows API.
